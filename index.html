09Pipeline Bottleneck — Nsight Systems Analysis

Tại sao per-cam FPS bị khóa ở ~0.6 fps dù tài nguyên thừa? Nsight Systems trace 54.3s cho câu trả lời.

Bottleneck = PGIE batch processing time, KHÔNG phải thiếu tài nguyên. GPU compute thực chỉ ~22% mỗi inference cycle, CPU avg ~50%, RAM 58% — nhưng pipeline stages xếp hàng sequential.
interval=1 → PGIE inference mỗi 2 batch (skip batch chỉ tracker, bypass buffer_process).
Inference batch: avg 3.8s (1.6-6.4s). GStreamer pipelining → effective output: ~1.5s/batch → 0.67 fps/cam.

Chuỗi pipeline thực tế (đo được)

RTSP 25 fps/cam × 32 cam = 800 fps ↓ NVDEC drop (DS_DEC_DROP_INTERVAL=3) = 200 fps (6.25/cam) ↓ Streammux batch=32 → ~6.2 batches/s input ↓ PGIE (interval=1: inference mỗi 2 batch) ← BOTTLENECK ↓ Inference batch: avg 3.8s (1.6-6.4s) — TRT + tracker + metadata ↓ Skip batch: tracker predict only (bypass buffer_process, không có trong NVTX) ↓ Back-pressure drop ~90% = ~20 fps total (lý thuyết 0.67, thực đo ~0.6/cam) ↓ Export gate (PGIE_EXPORT_FPS=2) = ~10 fps total (~0.30/cam) ↓ Pad probe → Orchestrator → VLM → Event
